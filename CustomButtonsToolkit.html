This button can be used to analize, edit, backup, publish and extract (from web pages) CustomButtos buttons. It contains features of several buttons thanks to the effort of several authors (Read the list of contributors for details).

Features

CB Toolkit is composed of two main bottom panels. The "Analize & Edit" panel and the "Backup & Publish" panel.

Analize & Edit panel description

Listed in this panel are the buttons that are placed on the various toolbars. The view is divided by three list boxes which can be reordered by name, by Id or by toolbar name by clicking on the corresponding button located on the panel toolbar (only in ascending order). Each line in this view is constituted by the button image, the button name, the button id and the name of the toolbar where the button is placed. Here it is possible to select only one button at a time.

Analyze bottom panel with lateral and bottom sections hidden

Lateral section buttons (From top to bottom) - Edition Options.

  • Edit selected button (Left click) + Toggle button initialization (Right Click).
  • Delete selected button (Left click) + Delete to palette (Right Click).
  • Clone selected button.
  • Copy selected button to clipboard.
  • Bookmark selected button.

Bottom section buttons (From left to right)

  • Analyze selected button's URI.
  • Analyze selected button's code tab.
  • Analyze selected button's initialization tab.
  • Analyze selected button's help tab.
  • Analyze selected button's attributes.
  • Analyze selected button's properties.
  • Inspect selected button with DOM Inspector extension (Left Click) + Inspect CustomButtons overlay with DOMi (Right Click). This button is shown only if DOMi is installed and enabled.
  • Switch to Backup & Publish panel button.

Backup & Publish panel description

In this window are all the buttons that are installed in the current profile listed in three columns. Each line in this view is constituted by the button image, the button name, the number id and the name of the toolbar where the button is placed (N/P stands for Not Placed). At the same time, each item displays a tooltip with some more details (full button id, toolbar id and the author's name (as long as the creator has set that attribute)). Here it is possible to select more than one button at a time.

Backup window with lateral and bottom sections hidden

Mouse Clicks on listboxes:

  • Left Click: Toggle single item selection.
  • Middle Click: Toggle all items selection.
  • Ctrl + Left Click on any listbox: Toggle that entire listbox item selection.

Lateral section buttons (From top to bottom).

  • Copy selected button's URI to clipboard. 2 possible outputs.
  • Copy selected button's URI as BBCode to clipboard. 3 possible outputs.
  • Copy selected button's URI as HTML to clipboard. 3 possible outputs.
  • Copy selected button's Image to clipboard. 3 possible outputs.

Bottom section buttons (From left to right).

  • Export selected buttons as HTML. 3 possible outputs.
  • Export selected buttons as XML. 2 possible outputs.
  • Backup CustomButtons overlay.
  • Import XML file as a new button.
  • Save selected button's images. Duplicated images will be ignored.
  • Switch to Analyze & Edit window button.

Features shared by both windows.

Both sections (lateral and bottom) of each window can be hidden to have more space to display the listed buttons. The actions that can normally be run from a button in any section can also be executed from the contextual menu (Right click). Buttons marked with a plus sign (+) can perform more than one action (Read tooltips of those buttons for details).

Advanced options window

Each option is self-explanatory. Read the tooltips for more details.



Button clicks/hotkey actions

There are a total of 10 button clicks, plus a hotkey that can be configured to run a total of 16 different actions.

Possible button clicks/hotkey actions are:

  • Open Toolkit Bottom Panel.
  • Export all installed buttons as HTML (Plain).
  • Export all installed buttons as HTML (With code preview).
  • Export all installed buttons as HTML (With code preview + Syntax highlighting).
  • Export all installed buttons as XML (Plain).
  • Export all installed buttons as XML (With code preview).
  • Import button from XML file.
  • Extract all buttons from page & open in editor.
  • Extract all buttons from page & Save as HTML (Plain).
  • Extract all buttons from page & Save as HTML (With code preview).
  • Extract all buttons from page & Save as HTML (With code preview + Syntax highlighting).
  • Extract all buttons from page & Save as XML (Plain).
  • Extract all buttons from page & Save as XML (With code preview).
  • Backup CustomButtons overlay.
  • Open the export folder location.
  • Open the profile folder.

Changelog

Updated December 17, 2014

  • Fixed: After I changed to a "Bottom panel" desing, the button "broke" on Thundebird (The panel would not open, although the other button functions worked properly). Fixed that.
  • Fixed: Exported HTML files couldn't display inline images from illegal sources, like "chrome://" URLs (chrome://branding/content/icon16.png) or "moz-icon" images (moz-icon://.png?size=16). I have "circumvented" this problem by converting all inline images inside exported HTML files to Base64.
  • Changed: Now button images can be "downloaded" inside the CB export folder location and will not "polute" the download manager with an entry of each image "downloaded".
  • Changed: Switched SyntaxHighlighter for Highlight.js. Reason: Highlight.js is smaller, faster and it has better highlighting (the latter is just my opinion).
  • Changed: Rewrited check for updates system. Now is possible to auto-check for updates every x amount of days with a delay of seconds after application startup.
  • Removed: Option to chose this button image size. More annoying than useful. It's easy enough to change the image of any CustomButtons button.
  • General enhancements of the kind "do more with less code".

Changelog - Updated July 4, 2014

This update is somewhat critical.
  • If you are using SeaMonkey, this update of CustomButtons Toolkit is MANDATORY. Using a version prior to July 4, 2014 of this button could mean that you can lose/ruin your CustomButton buttons. This is because now with the 2.26.x version of SeaMonkey, the window called Analyze & Edit (in the new version of this button, the window becomes a bottom panel), shows also the buttons that are not placed on any toolbar (which should not happen). If you try to edit those buttons, there is the risk of ruining them.
  • If you are using Firefox version 29 or above or any other browser based on Firefox 29 or above, CustomButtons Toolkit will not display buttons placed in certain toolbars (for example, buttons placed inside the AppButton). This is because starting from this version (29), Firefox has implemented a new way to create and manage buttons and toolbars. In this case, the update of CustomButtons Toolkit is not critical, but it is recommended.
  • If you are using Palemoon 24.x.x, Firefox ESR 24.x or any other browser based on prior versions of Firefox 28 (for example, Waterfox 28), you are safe for now and you do not need to update CustomButtons Toolkit. But know this, I have completely abandoned the design of "windows" for the CustomButtons Toolkit interface in favor to a "bottom panel design". So I'm not going to check if older versions of CustomButtons Toolkit will continue working properly or not in these browsers.

Compatibility

This button is intended to be cross-platform (Linux/Mac OS/Windows), %100 Unicode compatible and to work with all Firefox-based browsers, SeaMonkey and Thunderbird. But it has been tested (extensively) only in the following environments.

Arch Linux (XFCE and KDE) - Firefox 34.x
Windows 7 x64 - Palemoon x64 25.x.x, Thunderbird 31.x, Cyberfox x64 34.x and latest versions of Firefox ESR/Stable/Beta

Not compatible with Komposer and SunBird.


Technical notes

  1. All options that require an external editor, use the same editor configured to edit CustomButtons buttons externally. In the event that this preference (extensions.custombuttons.external_editor) has not been set, CB Toolkit will ask if you want to set an external editor on first use of any option that requires it. Reason: The easiest way I've found for this feature to be multiplatform.
  2. The styles of both windows were not created for aesthetic reasons. Reason: In certain environments with certain font types and/or sizes, the button labels can become way too big to fit inside the buttons themselves. Having the styles enabled corrects this behavior. The icons on buttons were added for usability reasons.
  3. Not compatible with Komposer and SunBird. Reason: Basically because these two programs are ancient. Trying to make this button compatible with any of these programs would require hundreds of conditions through the CB Toolkit script. And in some cases, I would be forced to drop the Unicode support, and that's not going to happen.

Error report

If any errors are found, report it ASAP on this button homepage with as much details as possible.
  • Wich plattform and its version (Linux, Mac or Windows)
  • Wich application and its version (Any Firefox-based browser, Thunderbir or SeaMonkey)
  • Error/Browser console output, if any.
  • If it's easier for you to report an error with images, and if the image is too large or there are more than one, publish the links to them, do not post them using the IMG tag in the message.
  • Be willing to follow troubleshooting instructions.
