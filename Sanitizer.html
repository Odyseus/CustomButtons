Install this button

Sanitizer is based/build around the script called sanitize.js (chrome://browser/content/sanitize.js) that can be found in all Firefox-based browsers (in SeaMonkey (resource://gre/modules/Sanitizer.jsm)). Hence the name of this button. Each menu item has the exact same name (except the ones that are abbreviated) and runs the exact same action as its equivalent in the browser's native clean dialog.

The reason I created this button? Basically, I find the native dialog to clear the history completely annoying. Starting with the way it has to be accessed, and even the way in which we have to choose the items to delete. With Sanitizer I wanted to achieve a simple goal, Click and Clear.

Sanitizer can be configured with a wide range of options to suit almost any need and/or taste. So it is advisable to read the tooltips for each option in the settings window before enabling or disabling options. But rest assured that the safest/less annoying options are ENABLED by default. And the most dangerous/more obtrusive ones are DISABLED by default.

Changelog

Changelog - May 24, 2014

  1. Added: SeaMonkey compatibility.
  2. Added: Sanitizer preferences can now be exported, imported and restored to their defaults. Accessible from the options window, bottom left corner. When exporting, the resulting file will be saved on your desktop.
  3. Added: New option to set a delay in milliseconds before the tooltip for this button is shown.
  4. Fixed: Finally fixed an annoyance that, under certain circumstances, the menu would not open on mouseover.
  5. Optimization: General code optimization.

Changelog - October 20, 2013

  1. Removed: "Run Clear All Data with Middle click". Reason: Read point 3.
  2. Removed: "Clear Saved Passwords" menu item. Reason: I want to keep the functionality of this button as similar as possible to the native dialog.
  3. Added: 6 new button clicks fully configurable. Each button click can be configured to run 7 different actions ("Open Options", "Clear All Data", "Clear All Data (Only Visible menu items)", "Clear All Data and Restart", "Clear All Data and Restart (OV)", "Clear All Data and Exit" and "Clear All Data and Exit (OV)"). Every action in turn can be configured to clear between certain amount of time (6 different time spans) and what to do with the currently opened tabs (6 different tabs behavior). Read the tooltips of these options in Sanitizer's settings window for more details.
  4. Added: The ability to configure a hotkey to perform any desired action. Read the tooltip of these option in Sanitizer's settings window for more details.
  5. Added: A fail safe so that it is NOT possible to use this button in Seamonkey or Thunderbird. For the simple fact that SeaMonkey does not use the sanitize.js script. And the sanitize script inside Thunderbird only has the ability to delete cookies and cache. It's too much trouble to make this button supports Thunderbird for only two options.
  6. Changed: The way Sanitizer creates its own preferences. Now it is possible to have an unlimited number of buttons, and all of them will have their individual preferences. Read the warnings below.
  7. Optimization: General code optimization.

Compatibility

This button is intended to work with all Firefox-based browsers and SeaMonkey. But it has been tested only in the following environments.

Arch Linux (XFCE and KDE): Firefox Stable 29.x
Windows 7 x64: Palemoon x64 24.x, Cyberfox x64 28.x , SeaMonkey 2.26 and latest versions of Firefox ESR/Stable/Beta

Not compatible with Thunderbird.

Error report

WARNING!! Read carefully the contents of the Readme tab in the configuration window of this button.

WARNING!! Do NOT update a version prior to October 20, 2013 of this button with an URI (a link) or code from a new version or you WILL end up with a lot of unused preferences. The correct way to update this button if you use a version prior to October 20, 2013 is to delete the old version and install the new one. Starting from the October 20, 2013 version, it does not matter the order in which these actions are performed (Read point 6 of October 20, 2013 changelog). On the other hand, if you're already using the October 20, 2013 version, you can safely upgrade to the May 24, 2014 version. The only thing to do after upgrading is open Sanitizer options and click the Accept button.

WARNING!! It is totally NOT advisable to have more than one version of this button installed. This is a heavy button (almost 1200 lines), and having more than one version installed at the same time can significantly slow down your browser.

If any errors are found, report it ASAP on this button homepage with as much details as possible.
  • Wich plattform and its version (Linux, Mac or Windows)
  • Wich application and its version (Any Firefox-based browser, Thunderbir or SeaMonkey)
  • Error/Browser console output, if any.
  • If it's easier for you to report an error with images, and if the image is too large or there are more than one, publish the links to them, do not post them using the IMG tag in the message.
  • Be willing to follow troubleshooting instructions.
